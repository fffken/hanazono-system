# バージョン管理ルール

Version: 1.0.0
Last Updated: 2025-05-03

## バージョニング規約 [v1.0.0]

### 1. セマンティックバージョニング
- バージョン形式: `vX.Y.Z`
  - X: メジャーバージョン (互換性を破壊する変更)
  - Y: マイナーバージョン (後方互換性のある機能追加)
  - Z: パッチバージョン (バグ修正のみ)

### 2. ファイルレベルのバージョン管理
- 各ファイルにはバージョン情報と最終更新日を記載

### 3. ドキュメント内のバージョン管理
- 各セクションにバージョン番号を記載

## Git 運用ルール [v1.0.0]

### 1. ブランチ命名規則
- `main`: 最新の安定バージョン
- `develop`: 開発用メインブランチ
- `feature/[機能名]`: 新機能開発
- `fix/[問題ID]`: バグ修正
